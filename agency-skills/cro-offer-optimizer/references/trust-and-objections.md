# Trust & Objections

## Objection mapping method
List the 3–5 reasons this trade's customers hesitate (ask the owner: "what do people ask before they book?"). For each: WHERE along the conversion path is it answered? An objection answered on a page the visitor never sees is unanswered.

## Standard local-service objection set (adapt per trade)
| Objection | Answer pattern | Placement |
|---|---|---|
| "Too expensive / afraid to ask" | Price transparency block (band or factors) | Service page, near CTA |
| "Can I trust strangers at my house / with my dog?" | Faces + names + years; licensed/insured/background-checked line | Trust strip + about; photo near form |
| "Will they actually show up?" | Response-time promise; scheduling reality; reviews mentioning punctuality | Near every CTA; proof section |
| "Is the work any good?" | Real job photos, specific reviews, guarantee/warranty terms | Proof sections between explanation and CTA |
| "Will I get spammed/hard-sold if I ask?" | "No-obligation" + what-happens-next script | On/above the form |

## Trust-signal hierarchy (strongest first)
1. Real reviews with reviewer names + specifics (quoted verbatim, sourced)
2. Real photos: owner, crew, actual local jobs
3. Numbers with receipts: years, jobs completed, response time
4. Credentials: license #, insured, certifications (verifiable ones)
5. Guarantees with actual terms
6. Local markers: town names, recognizable job locations (verified only)
Stock photos, unverifiable badges, and "award-winning" without the award are anti-signals — they read as compensating.

## Placement rule
Trust interleaves with action: hero → trust strip → explanation → proof → CTA → deeper proof → CTA. Don't quarantine all trust on an About page — the skeptic converts on the page they're already on.
