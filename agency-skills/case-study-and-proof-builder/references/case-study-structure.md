# Case Study Structure

Target length: 400–700 words for the full version. Owner-readable — a busy contractor should get the point skimming headers.

1. **Header:** Client name (or anonymized descriptor), trade + town, one-line result ("From invisible on Google to 40 tracked calls a month").
2. **The situation (2–3 sentences):** The business problem in the owner's own words from the interview. No tech jargon. "Customers said they couldn't find the number" beats "suboptimal NAP consistency."
3. **What was in the way (3–5 bullets):** Concrete findings — no mobile site, no tracking, thin service pages, unclaimed GBP. Pull from the original audit if one exists.
4. **What we did (4–6 bullets):** Deliverables in plain English, each tied to why it matters ("Built 22 town pages with real local detail — so nearby searches actually have a page to land on").
5. **What changed (the proof section):** Metrics with dates and sources per proof-rules. Include the screenshot(s). If early-stage, structural changes ARE the result ("every call and quote request is now tracked — before, nothing was measured").
6. **Owner's words (testimonial):** Verbatim quote, name, business.
7. **The takeaway + CTA:** One sentence generalizing to the reader's situation, then the Free Site Audit CTA.

## Derivative formats (produce in the same pass)
- **Proof block (3–4 sentences):** situation → work → verified result → client name. For offer sheets and the agency site.
- **Stat line (1 sentence):** the single strongest verified fact.
- **Social version (80–120 words):** first-person from Manny, no hashtag spam, ends with a soft CTA.
