# Proof Rules

## Allowed (with evidence in hand)
- Live builds and live URLs — link them; screenshot the before-state if a rebuild.
- Approved client names — approval must be explicit and noted; otherwise anonymize ("a Lexington pet grooming salon").
- Confirmed GA4 events — exact event names and counts, with the date range visible in the screenshot. Say "tracked calls" / "tracked quote requests," not "leads" unless the event maps to a real-lead definition per `lead-tracking-installer`.
- Real screenshots — GA4, Search Console, GBP insights, the site itself. Unedited except cropping/redaction.
- Testimonials — verbatim, with written permission (a text/email saying "yes you can use that" counts; save it).
- Structural improvements — pages built, schema added, tracking installed, load-time deltas, accessibility fixes. These are always safe because they're observable.
- Visibility data — Search Console impressions/clicks trends, framed as "search visibility," never as ranking claims.

## Not allowed
- Invented or extrapolated lifts ("estimated 40% more calls").
- Revenue attribution the client didn't confirm in writing.
- Ranking claims ("now #1 for dog grooming Lexington") — even if momentarily true, rankings fluctuate and the claim ages into a lie.
- Fake or paraphrased-as-quoted testimonials.
- AI visibility claims ("cited by ChatGPT/AI Overviews") unless screenshotted, dated, and framed as "was cited on [date]" — these change daily.
- Cherry-picked date ranges that manufacture a trend (comparing peak season to off season without saying so).

## Presentation rules
- Every metric: number + date range + source ("GA4, May 1–31, 2026").
- Before/after screenshots: same viewport, same zoom, labeled with dates.
- Modest true results outperform impressive vague ones in local-service sales conversations — owners smell inflation.
