# Page Blueprints

## Homepage (top to bottom)
1. **Hero:** H1 states service + area plainly ("Seamless Gutters, Cut On-Site — Serving the South Shore"). Subline: for whom / the differentiator. Primary CTA + phone. Real photo, not stock.
2. **Trust strip:** review stars + count, years, license/insured badges — small, immediately under hero.
3. **Services block:** cards → service pages. 3–6, real services only.
4. **Proof section:** 2–3 verbatim reviews or a case-study block. CTA repeat.
5. **Service area:** towns as links to location pages (this is the anti-orphan link block).
6. **How it works:** 3 steps max (call → quote → done). Sets expectations, reduces call anxiety.
7. **FAQ teaser:** 3–4 top questions, link to full FAQ.
8. **Final CTA band** + footer (NAP exactly matching GBP, hours, license #).

## Service detail page
1. H1 = service + area. First 60 words answer "what is this and is it what I need" (answer-first — this is what AI search quotes).
2. Who it's for / when you need it (symptoms the customer recognizes).
3. What's included / how it works.
4. Pricing: band, starting-at, or honest "what affects the price" — never nothing.
5. Proof specific to this service if it exists.
6. FAQ block (3–5, schema-synced). CTA after sections 2, 5, 6.

## Contact / quote page
- Fields: name, phone, town, what do you need (textarea). NOTHING else without a fight. Every extra field costs conversions.
- Above the form: response expectation ("we reply same business day").
- Phone + text option beside the form — some people will never fill a form.
- Confirmation state: thank you + what happens next + GA4 event fires here (per lead-tracking conventions).

## Mobile rules (most local traffic is mobile)
- Sticky bottom CTA bar: Call / Text / Book (Golden Paws pattern) for phone-first businesses.
- Tap targets ≥ 44px; form inputs with correct `type=`/`inputmode=` (tel keyboard for phone).
- Test every blueprint at 375px width before calling it done.
